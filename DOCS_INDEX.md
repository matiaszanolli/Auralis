## üí° Documentation Guidelines

### When to Add Documentation
- **Major features** ‚Üí Create in appropriate `docs/` subdirectory
- **Breaking changes** ‚Üí Update `docs/deployment/VERSION_MIGRATION_ROADMAP.md`
- **Build changes** ‚Üí Update build guides in `docs/development/`
- **API changes** ‚Üí Update `CLAUDE.md` and `docs/api/`
- **UI changes** ‚Üí Update design docs in `docs/design/`

### When to Archive Documentation
- **Session summaries** ‚Üí Move to `docs/archive/progress-reports/`
- **Phase completions** ‚Üí Move to `docs/archive/phase-completions/`
- **Implementation reports** ‚Üí Move to `docs/archive/`
- **Outdated guides** ‚Üí Move to `docs/archive/` with date stamp

### Where to Document
- **Getting Started** ‚Üí `docs/getting-started/` (or `README.md` for now)
- **Architecture** ‚Üí `CLAUDE.md`
- **Development** ‚Üí `docs/development/`
- **Design** ‚Üí `docs/design/`
- **API** ‚Üí `docs/api/`
- **Deployment** ‚Üí `docs/deployment/`
- **Historical** ‚Üí `docs/archive/`
